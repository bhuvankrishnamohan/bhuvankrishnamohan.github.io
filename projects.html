  • Emergency Communicator (Click for video) - Computer Science Project Semester 1 - 12/4/23 - 12/21/23 Our app is an emergency alert poster. The app has a admin and client panel. When admin is clicked, the admin can choose between three options, earthquake, fire, and lockdown. When one is chosen, it will alert the admin and post the message on the API. When going to the previous reports page, it will use the “get” from the API to retrieve all the previous reports, and the delete button will delete all previous reports with a password from the API. When the client is clicked, it will show the most recent report. When reporting the type of emergency, other students and teachers will be able to see what is happening. This can also be known as “post”. My role in this was the developer, coding all the API and backend javascript for funcionality. The difficulties were getting it through the API and bypassing CORS Blocker. The debugging was the hardest part. I learned that there is always a way to make the code work. I just need to think in the right direction.
